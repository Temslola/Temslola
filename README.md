# Temslola

Application security tester focused on **web and API penetration testing**.

I test web, API, and mobile applications in a banking environment, and I build
small tools to make the mechanical parts of that work repeatable — so manual
testing time goes to business logic, where tools are weakest.

---

### 🔧 Projects

**[semgrep-appsec-rules](https://github.com/Temslola/semgrep-appsec-rules)** — Custom Semgrep rules for the vulnerability classes I find by hand: BOLA, insecure JWT verification, mass assignment, and injection. Every rule ships with vulnerable and safe test fixtures, validated in CI.
`Semgrep` · `SAST` · `OWASP API Top 10`

**[apisec-harness](https://github.com/Temslola/apisec-harness)** — Reads an OpenAPI spec and generates OWASP API Top 10 test cases: BOLA, broken authentication, mass assignment, and missing rate limiting. Tested against an in-memory fake API, so the suite runs with no network. Exits non-zero on high findings for CI gating.
`Python` · `OpenAPI` · `OWASP API Top 10`

**[pentest-report-gen](https://github.com/Temslola/pentest-report-gen)** — Turns structured findings into Markdown or HTML pentest reports, with CVSS v3.1 base scores computed from vector strings and verified against FIRST.org reference values.
`Python` · `CVSS 3.1` · `Jinja2`
